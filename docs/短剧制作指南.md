## 一、完整操作教程

### 整体流程总览（文字流程图）

从打开网页到下载成片，你会顺序经过下面这条流水线。把它当作一条「单向传送带」——前一步没做好，后面没法补：

```
打开网页
  │
  ├─ 0 准备：设置页填好 4 类 AI 服务（文本/图片/视频/音频）+ 录入音色库
  │
  ├─ 1 建剧：首页「新建项目」（填名称/集数/视觉风格）
  │
  ├─ 2 建集：项目页「添加集」并锁定 本集的 图/视频/音频 三套配置（锁定后不可改）
  │
  └─ 进入单集工作台（顶部进度条 pipelineProgress/11）：
        │
        ├─ 剧本面板（panel=script，5 步）
        │     02 原始内容 → 03 AI改写 → 04 提取角色与场景 → 05 分配音色 → 06 分镜列表
        │
        ├─ 制作面板（panel=production，6 个 tab，按顺序）
        │     07 角色形象 → 08 场景图片 → 09 镜头图片(宫格图工具) → 10 配音生成 → 11 视频生成 → 12 视频合成
        │     （07/08/09/10 可大致并行，但都要在 11/12 之前完成）
        │
        └─ 导出面板（panel=export）
              13 拼接导出 → 下载视频
```

> 术语速查：**镜头**（shot）= 视频里一个连续的画面段落，约 4-12 秒；**分镜**（storyboard）= 把整集剧本拆成一个个镜头的清单；**首帧/尾帧**（first/last frame）= 一个镜头开始/结束的那张静态图；**TTS**（Text-To-Speech）= 文字转语音配音。

---

### 第 0 步：准备（部署 + 设置页配 4 类服务）

#### 部署要点
- 后端默认端口 **5679**（`npm run dev`），前端默认 **3013**（`npm run dev`，会自动把 `/api` 和 `/static` 代理到 5679）。
- 合成/导出依赖 **ffmpeg**，且必须是**带 `--enable-libass`（支持 subtitles 滤镜）的版本**，否则字幕会被静默跳过——成片没字幕但流程不报错。
- 数据库和生成的素材都落在项目根目录的 `data/` 下：`data/huobao_drama.db`（SQLite）+ `data/static/`（图片/视频文件）。删除 `data/` 会清空所有项目数据。

#### 设置页要配的 4 类服务
打开网页 → 左侧菜单「设置」（`/settings`）→ 顶部「AI 服务」标签页。页面有 4 个固定区段，每个区段右上角点「+ 添加」会弹出配置表单，字段为：**配置名称 / 服务商 / 优先级 / API Key / Base URL / 模型（逗号分隔）**。

需要配齐 4 类：

| 服务类型 | 用途 | 必填 |
|---|---|---|
| **文本**（text） | 所有 AI Agent 用（改写/提取/分镜/宫格提示词） | 是 |
| **图片**（image） | 角色图/场景图/镜头首尾帧/宫格图 | 是 |
| **视频**（video） | 图生视频（镜头视频） | 是 |
| **音频**（audio） | TTS 配音 | 是 |

> 具体每类选哪个模型/填哪个官方 endpoint，见[第三节·模型推荐](#三模型选型推荐官方-api禁中转站)。本节只给操作要点：用户明确要求只用官方 API、不用中转站（如 chatfire / api.chatfire.site）。**特别注意**：UI 里 volcengine 视频/图像的默认推荐 baseUrl 可能是中转站，换官方要手动改 baseUrl 为 `https://ark.cn-beijing.volces.com`。

每个配置填完点底部「测试配置」按钮验证连通性。**4 条配置都标「启用」**才生效。

#### 另一个必做：录入音色库（否则配音会错配）
设置页切到「音色」标签页（对应数据库 `ai_voices` 表），录入你 TTS 服务商的真实音色。**每条要填**：voiceId、voiceName、description（JSON 数组，建议格式：`["性别","音色特质1","音色特质2","适合角色类型1","适合角色类型2"]`）、gender、language。

> 为什么必做：如果 `ai_voices` 表是空的，系统会回退到 6 个写死的 OpenAI 默认音色（alloy/echo/...），但你的 TTS 是 minimax——这些音色 ID 在 minimax 里不存在，**合成时会失败或用错音色**（这是已知问题 #85）。先把 minimax 官方音色录全，是治本。

#### Agent 配置（可选但推荐）
设置页底部勾选「Agent 高级配置」开关才会出现「Agent 配置」和「Skills」标签页。这里可以**不改代码就调提示词**（对应 `agent_configs` 表）：比如想让分镜镜头短一点，就去 `storyboard_breaker` 的 systemPrompt 里加「每个镜头 4-6 秒，节奏快，首镜必须有视觉钩子」。

> 术语：**Agent**（智能体）= 一个会读剧本、调工具、写数据的 AI 助手。火宝有 5 个：改写、提取、配音、分镜、宫格提示词。

#### 完成判断
4 类 AI 服务都启用 + 音色库至少录了几条 minimax 音色 +（可选）Agent systemPrompt 调过 → 可以进下一步。

#### 常见卡点
- 测试配置失败：多半是 API Key 错、或 baseUrl 带了多余路径。每个服务商的端点前缀不同（openai/minimax 是 `/v1`，gemini 是 `/v1beta`，volcengine 是 `/api/v3`，ali 是 `/api/v1`，vidu 是 `/ent/v2`），UI 会自动拼接，别手动在 baseUrl 里加。
- 音色没录全 → 第 6 步分配音色时会撞 #85。

### 第 1 步：建剧

#### 位置 / 按钮
首页（`/`，`index.vue`）→ 点「新建项目」→ 弹窗填：项目名称 / 计划集数 / 视觉风格 → 「创建」。

#### 操作要点
- **视觉风格**会作为整剧基调（影响后续图片 prompt）。常见的有「电影感写实 / 国风工笔 / 二次元 / 写实短剧」。
- 集数可以后加，不必一次填满。

#### 产出
- 数据库 `dramas` 表一条记录。
- 创建后自动跳到项目页（`/drama/[id]`），侧边栏列出所有集。

#### 完成判断
项目页能看到你刚建的剧，进去后是空的集列表 → 进第 2 步。

#### 常见卡点
- 项目名重复不影响功能，但建议起一个能区分的名字。

---

### 第 2 步：建集 + 锁定配置（非常关键）

#### 位置 / 按钮
项目页（`drama/[id]/index.vue`）→ 点「添加集」→ 弹窗里**为本集选 3 套配置：图片 / 视频 / 音频**→ 「创建并锁定配置」。

#### 操作要点
- **锁定的意思**：本集后续所有图片/视频/配音都走你选的这 3 套配置，**一旦创建就不能改**。所以选的时候要想清楚——尤其视频模型决定了镜头时长风格（Seedance 默认 `doubao-seedance-1-5-pro-251215`，时长会被归一到 4-12 秒）。
- 如果不同集想用不同模型，就分批建集、分别锁定。
- 视频配置如果选了 vidu，注意它靠 Webhook 回调、不支持轮询，新手不建议。

#### 产出
- `episodes` 表一条记录，关联了 imageConfigId / videoConfigId / audioConfigId。
- 进入该集工作台后，每个制作 tab 顶部都会显示「当前集锁定模型」标签（`lockedImageConfigLabel` / `lockedAudioConfigLabel`），提醒你别想换。

#### 完成判断
项目页能看到这一集，点进去是空的单集工作台（顶部进度 0/11）→ 进第 3 步。

#### 常见卡点
- 想中途换模型？只能删集重建（数据会丢），或新建一集重新跑。**这就是为什么「锁定」要慎重**。

### 进入单集工作台

点击某 一集进入 `drama/[id]/episode/[episodeNumber].vue`（这是整个流水线的主战场，4313 行的主工作台）。页面结构：
- **顶部 topbar**：进度条 `pipelineProgress/11` + 「刷新」+ 主按钮（动态变「开始制作 / 继续制作 / 查看成片」）。
- **左侧侧边栏**：分 3 组——**剧本**（5 步）/ **制作**（5 项）/ **导出**。每项前面有个圆点，绿色 = 已完成（`done`）。
- **主内容区**：根据当前阶段切换 `panel`（script / production / export）。
- **底部气泡导航**：「上一步」+ 步骤圆点 + 「下一步」。

顶部点「开始制作」进入剧本面板。

### 第 3 步：原始内容（剧本·01）

#### 位置 / 按钮
剧本面板，步骤 01「原始内容」。把小说原文 / 剧本大纲 / 分镜描述粘贴进大文本框 → 点「保存」。

#### 操作要点
- 这里存的是**原始素材**（数据库 `episodes.rawContent`）。AI 改写会读它。
- 内容长度建议：一集对应约 1-3 分钟成片，原始文本约 800-2000 字比较合适。太长会被压缩、太短会被扩写。

#### 产出
- `episodes.rawContent` 写入。

#### 完成判断
文本框有内容 → 侧边栏「原始内容」圆点变绿（`done`）→ 底部「下一步」可点。

#### 常见卡点
- 直接粘长篇小说整本 → AI 会试着压成一集，效果差。建议**按集切段**再粘。

---

### 第 4 步：AI 改写（剧本·02）

#### 位置 / 按钮
步骤 02「AI 改写」。两个按钮：「开始改写」或「跳过改写」。

#### 操作要点
- 「开始改写」调 `script_rewriter` Agent，把原文整理成**格式化剧本**。规范格式长这样：
  ```
  ## S01 | 内景 · 咖啡厅 | 黄昏
  （动作描写）
  小红：（微笑）等很久了吗？
  ```
- 「跳过改写」= 直接拿原文进入提取（适合你原文已经是剧本格式的情况）。
- 改写需要时间（一次完整 LLM 调用，不流式），耐心等。

#### 怎么判断改写是否正确
对照下面清单自查（判断标准细则见[第二节·第 1、3 节](#1-提取的角色与场景怎么判断对不对)）：
- [ ] **每个场景都有场景头**：`## S编号 | 内/外景 · 地点 | 具体时段`。时段要写「黄昏/深夜/清晨」这种具体的，不要笼统写「白天」。
- [ ] **场景编号连续**（S01、S02、S03…不跳号）。
- [ ] **每场约 30-60 秒**（按字数约 80-150 字）。过长 = 没拆场。
- [ ] **对白格式统一**：`角色名：（表情）台词`。
- [ ] **没有镜头语言**（特写/推镜/远景这些不该出现，那是分镜的事）。出现 = 跑偏。
- [ ] **字数比原文增 20-30%** 算正常。翻倍 = 过度扩写，几乎不变 = 偷懒。

#### 人工纠正
- 直接在文本框里手动改（前端支持编辑），保存即生效。
- 或在对话框写指令重跑，比如「重写，要求：每场控制在 40 秒内、对白用 `角色名：（表情）台词` 格式、不要旁白」。
- 注意：重跑会**全量覆盖** `scriptContent`，手动改的内容会被冲掉，先备份。

#### 产出
- `episodes.scriptContent` 写入。

#### 完成判断
`scriptContent` 有内容 → 圆点变绿 → 「下一步」可点。

#### 常见卡点
- Agent 偷懒只存原文（检查 `scriptContent` 和 `rawContent` 是否几乎一样）。
- 漏角色（原文里的配角被省略，后面提取会跟着漏）。

### 第 5 步：提取角色与场景（剧本·03）

#### 位置 / 按钮
步骤 03「提取角色与场景」。点「开始提取」。

#### 操作要点
- 调 `extractor` Agent，从剧本里抽出**角色**（存 `characters` 表）和**场景**（存 `scenes` 表），并关联到本集。
- **去重逻辑**：角色按 `name` 字符串精确匹配（同名 = 合并更新，不同名 = 新增）；场景按 `location + time` 匹配。**同地点不同时段 = 不同场景**（比如「咖啡厅@黄昏」和「咖啡厅@深夜」是两条），这是对的，因为光线完全不同。

#### 怎么判断提取是否正确
**角色**：
- [ ] 没有遗漏（所有有台词、有重要动作的角色都在）。
- [ ] 没有重复（同一角色不该有两条，除非是「幼年/成年」等明确阶段）。
- [ ] **`appearance` 字段详尽**（300-500 字外貌描写：性别/年龄/体型/面部/发型/着装）。**这个字段最重要**——它直接决定后面角色图的一致性。空白或一句话 = 不合格，手动补。
- [ ] `personality` 有标签（沉稳/张扬…），`role` 定位明确（主角/配角）。

**场景**：
- [ ] 每个场景 `location + time` 唯一。
- [ ] `prompt` 字段是英文且纯背景（**用于 AI 图片生成，不含人物**）。如果写成中文或带人物，手动改。
- [ ] 没有凭空创造（每个都能在剧本里找到）。

#### 人工纠正
- **新增角色/场景**：在前端面板手动加。
- **修改外观**：直接编辑 `characters.appearance`（**最该手动精修的字段**）。
- **同名合并误伤**（比如「老王（父亲）」和「老王（邻居）」被并成一条）：手动改 `name` 区分。
- 重跑 extractor 时，空字段不会清空你已填的原值（有保护逻辑），所以可以放心重跑。

#### 产出
- `characters` / `scenes` 表 + `episode_characters` / `episode_scenes` 关联。

#### 完成判断
角色列表非空 → 圆点变绿 → 「下一步」可点。

#### 常见卡点
- 漏提配角 → 后续分镜 `character_ids` 无可选，重跑并点名「必须包含 XX」。
- 场景时段不统一（一次写「黄昏」、下次写「傍晚」）→ 同地点被分裂成两条，统一措辞。
- `appearance` 太短 → 角色图一致性差，务必手动补长。

### 第 6 步：分配音色（剧本·04）

#### 位置 / 按钮
步骤 04「分配音色」。点「AI 自动分配」。已分配后显示「重新分配」。旁边有「生成试听文件」，单角色卡片有「生成试听 / 重新试听」。

#### 操作要点
- 调 `voice_assigner` Agent，为每个角色匹配一个音色（写入 `characters.voiceStyle` = 音色 ID，`characters.voiceProvider` = 音色服务商）。
- 音色库来自 `ai_voices` 表（你第 0 步录入的）。如果表空，会回退到 6 个 OpenAI 音色——但 minimax TTS 用不了这些，**这是 #85 错配的根因**。
- **强烈建议**：分配后先对主角逐个「生成试听」，听一遍再批量，避免浪费配额。

#### 怎么判断音色是否合理
- [ ] **每个角色都分到**了（列表里没有「未分配」）。
- [ ] **性别匹配**：男角 → 男声，女角 → 女声。（系统的性别推断是正则关键词匹配，可能不准，人工核对。）
- [ ] **音色 ID 真实存在**于你的音色库（去设置页 `ai_voices` 对照）。
- [ ] 主角辨识度高（不和配角撞音色）。
- [ ] 音色服务商 = 你 TTS 配置的 provider（minimax）。

#### 人工纠正
- 先把音色库录全是治本（第 0 步）。
- 手动改：前端角色卡片直接选音色，或编辑数据库。
- 重跑带约束：对话里写「为主角 XX 选 minimax 的 `novel-v7` 女声，写明理由」。

#### 产出
- `characters.voiceStyle` + `characters.voiceProvider`。

#### 完成判断
**所有角色都分到音色**（`charsVoiced === chars.length`）→ 圆点变绿 → 「下一步」可点。

#### 常见卡点
- 音色库没录 → 分配了 OpenAI 音色 → 合成失败。
- 跨集串味：`voiceStyle` 写在项目级的 `characters` 表，改一个角色会影响所有集，改之前确认。

### 第 7 步：分镜（剧本·05）

#### 位置 / 按钮
步骤 05「分镜列表」。点「AI 拆解分镜」。已有镜头显示「重新拆解」。还有「添加」可手动加空镜头。

#### 操作要点
- 调 `storyboard_breaker` Agent，把剧本拆成一个个镜头（`storyboards` 表）。
- Agent 会拿到本集锁定的视频模型名，按模型特性生成 `video_prompt` 和时长。
- 每个镜头字段很全：景别（shotType）/ 角度（angle）/ 运镜（movement）/ 地点 / 时段 / 绑定角色（character_ids）/ 动作（action）/ 对白（dialogue）/ 描述 / 结束画面（result）/ 氛围（atmosphere）/ 静态首帧提示词（imagePrompt）/ 动态视频提示词（videoPrompt）/ 配乐（bgmPrompt）/ 音效（soundEffect）/ 时长（duration）/ 关联场景（sceneId）。

> 术语：**景别**= 画面里人有多大（远景/全景/中景/近景/特写）；**运镜**= 摄像机动不动（固定/推/拉/摇/跟/移）。

#### 怎么判断分镜是否正确
- [ ] **镜头数合理**：30 秒场景约拆 2-3 个镜头，一集 60 秒约 4-6 镜。
- [ ] **`duration` 之和 ≈ 集总时长**（系统会自动算 `episodes.duration`）。
- [ ] **时长建议**：系统默认 10-15 秒/镜，但短视频要快节奏会显得拖。**想短**：去设置页改 `storyboard_breaker` 的 systemPrompt 加「每镜 4-6 秒」。
- [ ] **首镜有钩子**：第 1 镜的 `imagePrompt` 要有视觉冲击力（决定观众划不划走）。
- [ ] **`character_ids` 正确**：每个出场角色都绑定；空镜头传 `[]`。用了不存在的 ID 会直接报错（整批不存）。
- [ ] **`sceneId` 正确**：能匹配 scenes 表。凭空造会报错。
- [ ] **`video_prompt` 有分段**：每 3 秒一段，用 `<n>` 标记时间段，格式：
  ```
  0-3秒：<location>咖啡厅</location>，近景，<role>小明</role>低头看手机。<n>3-6秒：...
  ```
- [ ] **`description`（人读）≠ `video_prompt`（模型读）**，两者不能混。
- [ ] **对白与镜头对应**：`dialogue` 里的角色必须在 `character_ids` 里。

#### 人工纠正
- 改单镜：前端编辑，或在对话里说「把第 3 镜 duration 改 5，character_ids 改 [2]」。
- 整体重拆：**注意会先删光本集所有分镜再重存，无法撤销**。
- 调时长风格：改 `agent_configs.storyboard_breaker.systemPrompt`。

#### 产出
- `storyboards` 表 + `episodes.duration` 自动更新。

#### 完成判断
`storyboards.length > 0` → 圆点变绿 → 底部「下一步」文案变成「进入制作」→ 切到 production 面板。

#### 常见卡点
- 时长过长（默认 10-15s）→ 改 systemPrompt。
- `character_ids` 用了不存在 ID → 整批报错，回去核对角色列表。
- `sceneId` 凭空造 → 同样报错，回去核对场景列表。

### 第 8 步：角色形象（制作·角色形象 tab）

制作面板第一个 tab「角色形象」。按钮：「批量生成」或每张卡片的「生成」。

#### 操作要点

#### 完成判断
所有需形象角色都有图（`visualChars` 全绿）→ 进下一 tab。

#### 常见卡点

### 第 10 步：镜头图片（制作·镜头图片 tab）— 含宫格图工具完整走法

> 这是整个流水线最复杂、也最关键的一步。新手最容易在这里卡住。

#### 两个独立开关（先理解这个，否则会乱）

镜头图片 tab 顶部有**两个开关**，控制不同环节：

**开关 A：`gridMode`（宫格图工具面板里的模式 Tab）**
- 在哪：点「宫格图工具」打开弹窗，顶部三个 Tab。
- 控制什么：**走宫格批量出图时**，一张大图里每个格子是什么。
- 只在「用宫格出图」时生效。不用宫格，这个开关什么都不做。

**开关 B：`frameMode`（镜头列表区顶部下拉）**
- 在哪：镜头图片 tab 顶部，下拉框，选项「仅首帧 / 首尾帧」（值 `first` / `first_last`）。
- 控制什么：**镜头列表的 UI 和单镜出图入口**——`first` 只显示首帧缩略图 + 「首帧」按钮；`first_last` 额外显示尾帧缩略图 + 「尾帧」按钮。
- 视频生成阶段也读它：首+尾都有 → 视频自动用 `first_last` 补间模式；只有首帧 → `single` 单参考模式。

#### 宫格图三种模式（gridMode）对照

| 维度 | 首帧（first_frame） | 首尾帧（first_last） | 多参考（multi_ref） |
|---|---|---|---|
| 单格语义 | **一格 = 一个镜头的首帧** | 按 `i % 2` 交替：偶数格(行内左)=首帧，奇数格(行内右)=尾帧 | **所有格子 = 同一个镜头**的不同机位/构图 |
| 切分后写入 | 全部 `firstFrameImage` | 交替写 `firstFrameImage` / `lastFrameImage` | 全部追加到 `referenceImages` |
| 选镜头 | 多选 | 多选 | **单选**（只对一个镜头） |
| 格数 vs 镜头 | 格数 = 出几个镜头的首帧 | 格数应 ≥ 镜头数×2；不够会循环复用 | 格数 = 给同一镜头出几张参考图 |
| 主要目的 | **跨镜一致性最好**（最常用） | 配合图生视频首尾帧补间 | 给单个镜头补参考图，提升稳定性 |

#### 「仅首帧 vs 首尾帧」明确操作指引（用户重点痛点 6）

> 选择策略（什么时候用哪个 gridMode、什么场景适合首尾帧补间）见[第二节·第 7 节](#7-镜头图宫格图排版策略)。本节只给操作步骤。

**场景一：我只要每个镜头一张首帧（最推荐的新手起步）**
1. 镜头图片 tab 顶部 `frameMode` 选「仅首帧」（`first`）。
2. 点「宫格图工具」→ 弹窗 step 0 顶部 mode Tab 选「首帧」（`first_frame`）。
3. 宫格下拉选 `3x3`（≤9 镜）或 `4x3`（≤12 镜）。
4. 镜头多选列表点「全选」。
5. 点「生成提示词」→ step 1 看提示词预览 → 「生成宫格图」。
6. step 2 等生成 → 自动进 step 3。
7. step 3 切分分配：左侧大图每格对应一个镜头首帧，右侧表格把每格分给对应镜头的「首帧」类型。**未分配的格被忽略**。
8. 「切分并分配」→ step 4 完成 → 关闭。
9. 此时每镜头都有首帧 → 视频生成走 `single` 单参考即可出片。

**场景二：我要给某些镜头同时出首帧+尾帧（做补间运镜，如转身、走动、推门）**
1. **先按场景一把首帧铺好**（这一步是基础）。
2. 挑出需要补间的镜头（3-4 个）。
3. 镜头图片 tab 顶部 `frameMode` 切到「首尾帧」（`first_last`）——这样镜头列表才会显示尾帧位。
4. 点「宫格图工具」→ mode Tab 选「首尾帧」（`first_last`）。
5. **宫格布局强烈建议选 2 列**（如 `2x3` 或 `2x4`）——这样视觉上就是干净的「左首帧、右尾帧」。选 3 列或 4 列奇偶交替仍在，但视觉不再是左右成对，新手别用。
6. 镜头多选挑那 3-4 个目标镜头。
7. 生成提示词 → 生成宫格图。
8. step 3 切分分配：**偶数格（index 0,2,4…）必须分给首帧（`first_frame`），奇数格（1,3,5…）必须分给尾帧（`last_frame`）**。如果搞反，视频会把尾帧当首帧，补间会错乱。
9. 切分并分配 → 完成。
10. 视频生成时这些镜头自动走 `first_last` 补间，运动更顺滑。

> 注意：`first_last` 模式下，尾帧的 prompt 会用分镜里的 `action` / `movement` 字段。**这两个字段填得越具体，尾帧的动作感越准**（比如「转身推门」而不是「站着」）。

**场景三：某个关键镜头（主角特写、关键道具）视频效果不稳，补几张参考图**
1. mode Tab 选「多参考」（`multi_ref`）。
2. 镜头列表**单选**那个目标镜头（radio）。
3. 宫格选 `2x2`（4 张）——**别开太大**，因为参考图硬上限 6 张，出 9 张会被截断还挤掉角色/场景参考。
4. 生成 → 切分 → 每格分配为该镜头的「参考图」（`reference`）类型。
5. 视频生成时该镜头走 `multiple` 参考模式，模型有更多构图线索。

#### 宫格图工具 4 步弹窗完整走法

| Step | 名称 | 界面 | 关键按钮/动作 |
|---|---|---|---|
| 0 配置 | 模式 + 镜头选择 | mode Tab、宫格下拉（默认 3x3）、镜头多选（multi_ref 单选）、「全选/取消全选」 | 「生成提示词」。底部摘要显示自动算出的 `rows×cols` 和已选数 |
| 1 提示词预览 | 整图 `grid_prompt` + 每格 `cell_prompts` | 标签显示来源（AI生成 / 模板兜底） | 「上一步」「重新生成」「生成宫格图」 |
| 2 生成中 | 转圈 + 状态文字 | 轮询状态（超时回 step 0） | 自动跳 step 3 |
| 3 切分分配 | 左：大图带半透明覆盖格按钮（点格子高亮）；右：分页「格子分配」表（格/镜头/类型/当前绑定） | 每格选 镜头 + 帧类型（首帧/尾帧/参考图） | 「返回」「切分并分配」。**提示：未分配格子会被忽略** |
| 4 完成 | 打勾 +「X 格已分配」 | 「关闭」 | 关弹窗并 refresh |

> 宫格图历史存在浏览器 localStorage（key `huobao:grid:${dramaId}:${epId}`），下次可点「继续切割」回到 step 3。

#### 切分分配的硬知识（避免踩坑）
- **切分是纯像素等分**（无内容识别）：单格固定 960x540，整图 = 960×cols × 540×rows，按行列切。模型把两个角色画到一格中间、或一格画空了，切分照样按像素切，不会自动归位。
- **未分配的格被忽略**（不会写回分镜）。
- **一格画了两个镜头内容/粘连**：别硬分配，**重出整张图**（单格没法单独重生）。
- **一格画空/出现文字水印**：同样重出整图。回到 step 0 改 `custom_prompt` 加强约束（加 `no merged panels, exactly N visible panels, no text`），或换图片 provider。
- **cell prompt 改英文**：代码生成的 cell prompt 是中文「格1：」混合，**图片模型英文效果更好**，生成后手动改英文。

#### 6 张参考图硬上限（要知道）
系统在收集参考图（用于图片/视频生成时的人物/场景锁定）时，按**固定顺序**收集，到 6 张就截断：
1. 每镜头首帧 → 2. 每镜头尾帧 → 3. 每镜头镜头图 → 4. 每镜头参考图 → 5. 所有场景图 → 6. 所有角色图。

**后果**：多角色群像剧本（5 角色 + 3 场景 + 12 镜头）参考图极易爆 6 张。爆了之后**靠后的场景图和角色图会被丢弃**，导致 prompt 里写了「图片3=张三」但实际没传，模型只能凭名字猜脸，一致性下降。

**优化策略**：让主角的 `imageUrl` 更早入库（顺序按 DB id 升序）；删掉没用的镜头参考图、未用场景图，腾配额；`multi_ref` 别开太大（推荐 2x2=4 张）。

#### 完成判断
镜头图片徽标 `shotImgCount / sbs.length`，至少每镜头都有首帧（绿点）→ 进下一 tab。

#### 常见卡点
- 混淆 `gridMode` 和 `frameMode` → 记住：**gridMode 管批量出图策略，frameMode 管单镜入口和视频参考模式**，两者独立。
- `first_last` 模式奇偶格分错 → 视频补间错乱。
- 一致性差 → 多半是参考图爆 6 张或 `appearance` 太短。

### 第 11 步：配音生成（制作·配音生成 tab）

#### 位置 / 按钮
第三个 tab「配音生成」。「批量生成」或单条「生成配音」。已生成可重抽。

#### 操作要点
- 为「有对白」的镜头生成 TTS 音频。系统会自动忽略「环境音/无对白/纯音效」等占位台词。
- 音色取自 `characters.voiceStyle`（第 6 步分配的）。speaker + 纯文本命中忽略表就不生成。
- 卡内有 `<audio>` 直接播放试听。

#### 怎么判断/调整
- 听一遍，音色不对 → 回第 6 步改 `voiceStyle` 再重生（**注意**：合成时会复用已存的 `ttsAudioUrl`，改了音色要先把旧的删掉或失效才会重新生成）。
- 时长不对 → 改分镜的 `dialogue` 文本（拆句/合并）让 TTS 时长逼近视频时长。
- 详细调整手法（时长对齐、字幕、BGM）见[第二节·第 6 节](#6-配音试听--时长对齐)。

#### 产出
- `storyboards.ttsAudioUrl`。

#### 完成判断
所有有对白的镜头都生成了（`ttsGeneratedCount === ttsEligibleCount`）→ 进下一 tab。

#### 常见卡点
- 音色错配（#85）→ 音色库没录全，回第 0 步。
- 想换音色但没触发重新生成 → 清掉旧的 `ttsAudioUrl`。

### 第 12 步：视频生成（制作·视频生成 tab）

#### 位置 / 按钮
第四个 tab「视频生成」。「批量视频」或单卡「生成视频」。

#### 操作要点
- 图生视频，每镜头一段 mp4。
- **参考模式自动判定**：
  - 首帧 + 尾帧都有 → `first_last`（首+尾补间，运镜最顺滑）。
  - 首帧 + 额外参考图 → `multiple`。
  - 只有首帧 → `single`。
- 视频模型走本集锁定的 videoConfig（如 Seedance `doubao-seedance-1-5-pro-251215`），时长归一到 [4,12] 秒，默认 `generate_audio=true`、`ratio=adaptive`、`watermark=false`。
- 失败时卡片上 `prod-error` 显示真实错误。

> **adapter 坑提醒**：`generateVideo` 入库时若 `referenceMode` 缺省会写成字符串 `'none'`，三个 adapter 的分支都不命中=纯文生视频，**务必显式选 single/first_last/multiple**。

#### 产出
- `storyboards.videoUrl`。

#### 完成判断
所有镜头都有视频 → 进下一 tab。

#### 常见卡点
- 单镜视频失败 → **不要整集重跑**，逐镜重试（看错误信息）。
- Vidu 模型靠 Webhook 回调，新手不建议。

> 系统局限：视频生成**没有自动重抽/批量重抽优化**，失败的要手动逐个点。

### 第 13 步：视频合成（制作·视频合成 tab）

#### 位置 / 按钮
第五个 tab「视频合成」。「批量合成」或单卡「开始合成」。已合成显示「重新合成」。

#### 操作要点
- 把每个镜头的「视频 + TTS 音频 + 字幕」用 ffmpeg 合成一条成片。
- 「开始合成」按钮在视频未生成时是 disabled（必须先有视频）。
- 「批量合成」会轮询状态（每 3 秒，最多 120 次）。失败镜头数会 toast 报出，错误写进 `failedComposeMessages` 显示在卡上。
- 合成参数：`libx264 -preset fast -crf 23`，音视频用 `-shortest` 截断对齐。有字幕时烧录（白字 FontSize=20、黑描边 Outline=2）。
- **字幕前提**：ffmpeg 必须带 `--enable-libass`，否则字幕被静默跳过（成片没字幕但不报错）。

#### 产出
- `storyboards.composedVideoUrl`（文件在 `data/static/composed/{uuid}.mp4`）。

#### 完成判断
所有镜头都合成成功 → 进导出。

#### 常见卡点
- **#84 单镜失败卡整集**：哪怕只有 1 镜没合成成功，整集导出按钮会一直 disabled（因为导出要求 `composedCount === sbs.length`）。**对策**：每镜合成成功后立即备份 mp4；分批跑（按场景/段落），失败的先看 `composeError` 里的真实 ffmpeg stderr 再单镜重试；急需出片可临时改源码把判断改成 `>= N`（改代码，非配置）。
- 字幕没了 → ffmpeg 没编 libass，换带字幕滤镜的 ffmpeg 版本。
- 音画不对齐：`-shortest` 是硬截。TTS 比视频长 → 丢尾句；TTS 比视频短 → 尾段静音黑场。**宁可重生 TTS 调时长**，别硬裁。

> 系统局限：**不内置 BGM**。compose 只混「视频 + TTS + 字幕」三轨，且显式忽略 `bgm/背景音` 这类 speaker。要加 BGM 需外挂手动合入，现成代码没有这条路径。

---

### 第 14 步：整集导出（导出面板）

#### 位置 / 按钮
侧边栏最后一组「导出·拼接导出」→ 切到 export 面板。点「开始拼接」。完成后显示 `<video>` 播放器 + 「下载视频」。

#### 操作要点
- 「开始拼接」按钮在 `composedCount === 0` 时 disabled。
- merge 是异步的：点完立即返回 `merge_id`，前端每 3 秒轮询 `mergeAPI.status` 直到 `completed/failed`。
- concat 参数：`-f concat -safe 0` + 强制重编码（`preset medium -crf 23`，音频重采样到 48k/192k）+ `+genpts`（防音画不同步）+ `+faststart`（边下边播）。
- **横竖屏**：merge **不做任何分辨率/比例转换**，成片方向完全继承自上游图生视频输出（Seedance 默认 `adaptive`）。要做 9:16 竖屏，必须在上游图片/视频生成阶段就锁死比例。
- 字幕只在单镜烧录，merge 不再叠加。

#### 产出
- `episodes.videoUrl` + `video_merges.mergedUrl`（文件在 `data/static/merged/{uuid}.mp4`）。

#### 完成判断
拼接完成 → 播放器能播 → 点「下载视频」存到本地。

#### 常见卡点
- 拼接 fail → 多半是某镜没合成（#84），回去看 compose 状态。
- 成片方向不对 → 上游图片/视频比例没锁，merge 补不了。

### 附：新手最容易踩的 5 个坑（系统局限总览）

1. **#84 单镜失败卡整集**：导出要求所有镜头都合成成功，1 镜失败 = 全卡。对策：逐镜重试 + 备份。
2. **#85 音色错配**：音色库（`ai_voices`）没录全，分配到不存在的音色。对策：第 0 步先录 minimax 音色。
3. **提示词部分写死**（#93 真相）：不是代码全写死，是 `agent_configs` 表默认空、回退到 `DEFAULT_PROMPTS`。**正解**：去设置页改 `systemPrompt`，不要改代码。但工具内部的格式字符串（如 `consistent art style`、`duration || 10`、`inferGender` 正则）是真写死，改需改代码。
4. **无自动重抽**：图片/视频不满意只能手动逐个点，没有「批量优化」按钮。
5. **无 BGM 内置**：成片只有人声 + 字幕，要音乐得外挂。

## 二、实战经验：判断标准与调整手法

> 本节只讲判断标准与调整手法。所有结论都基于代码逐行核对，标注了真实落点字段和按钮位置。

### 1. 提取的角色与场景：怎么判断对不对

#### 判断标准清单

**角色表**

- [ ] **零遗漏**：剧本里所有有台词、有重要动作的角色都在列表里（一次性龙套可忽略）。漏一个，后面分镜的 `character_ids` 就无角色可选。
- [ ] **零误并**：同一角色只有一条；除非是「幼年/成年」这种明确不同阶段。**去重是按 `name` 字符串硬匹配**（`extract-tools.ts:140`），所以「老王（父亲）」和「老王（邻居）」被并成一条就是 bug。
- [ ] **`appearance` 字段 300–500 字**：必须覆盖性别、年龄、体型、面部特征、发型、着装、配饰。这是直接决定角色图一致性的字段，**一句话写完=不合格**。
- [ ] **`personality` 有标签**：如「沉稳内敛 / 外向张扬」。
- [ ] **`role` 定位明确**：主角 / 配角 / 龙套。

**场景表**

- [ ] **每个场景 `location + time` 组合唯一**：「咖啡厅@黄昏」和「咖啡厅@深夜」应是两条（光线完全不同，这是对的）。
- [ ] **`prompt` 是英文且纯背景**（SKILL 强制：不含人物）。中文 prompt 或带人物描述的，要去手改。
- [ ] **没有凭空创造**：每个场景都能在剧本里找到对应文字。
- [ ] **时段统一**：同地点别一会「黄昏」一会「傍晚」，否则被分裂成两条。

#### 调整方法（具体到字段）

| 症状 | 改哪个字段 / 点哪 | 怎么改 |
|---|---|---|
| 漏角色 | 前端「角色与场景」面板手动新增；或重跑 extractor 对话里点名「必须包含 XX」 | 新增一行 `characters` 记录，关联到本集 |
| 角色被误并 | 手动改 `characters.name`，加区分后缀如「老王（父亲）」 | **重跑 extractor 不会清空空字段**（`char.role \|\| existing.role`），手动填的内容安全 |
| `appearance` 太短 | **直接编辑 `characters.appearance`**，手写到 300+ 字 | 这是全流程最该手动精修的字段，没有之一 |
| 场景 `prompt` 是中文 / 带人物 | 编辑 `scenes.prompt`，改英文纯背景描述 | 例：「雨夜咖啡厅，霓虹招牌反光湿润路面，暖黄台灯，无人」 |
| 角色存在但本集不显示 | 重跑 extractor，会自动 `linkCharToEpisode` | 不用删了重建 |
| 场景时段被分裂 | 统一 `scenes.time` 措辞（全用「黄昏」或全用「傍晚」） | 删掉重复条 |

#### 反例 vs 正例

**反例（外观）**：「张三，男，30 岁，穿西装。」——一句话。出图时每张脸都不一样，西装颜色每次变。

**正例（外观）**：「张三，男，30 岁，身高 180，瘦削长脸，浓眉单眼皮，左眉骨一道 3cm 旧疤，板寸黑发，络腮短须，深灰双排扣西装内搭黑高领，左手戴银色机械表，右耳一枚小银钉。气质阴郁沉稳。」——300 字锚死。出图一致性立刻翻倍。

### 2. 音色分配：合不合理 + #85 兜底

#### 判断标准

- [ ] **每个角色都分配了**：列表无「未分配」。DEFAULT_PROMPTS 强制不遗漏。
- [ ] **性别匹配**：男角男声、女角女声。**`inferGender` 是正则启发式**（`voice-tools.ts:101`），音色名不带性别词会判成「中性」，必须人工核对。
- [ ] **`voice_id` 真实存在**：去「设置 → 音色管理」对照 `ai_voices` 表确认。
- [ ] **音色 provider = minimax**：系统 TTS 只支持 minimax，`ai_voices` 表空时会兜底到 OpenAI 的 6 条音色（alloy/echo 等），**合成时必然失败**——这是 #85 最常见的根因。
- [ ] **主角辨识度**：主角不和配角撞音色。
- [ ] **跨集一致**：`voiceStyle` 写在项目级 `characters` 表，改一个角色会影响所有集。

#### 调整方法

1. **治本第一步**：去「设置 → 音色（ai_voices）」**录入 minimax 官方音色**：voiceId / voiceName / description（JSON 数组）/ gender / language。表空 = 必崩。
2. **手动改 `characters.voiceStyle`**：前端角色卡片直接选；或改 DB。
3. **重跑带约束**：对话框写「为主角 XX 选 minimax 的 `novel-v7` 女声，理由要写明」。
4. **试听再批量**：分配后先用 TTS 工具试听一句对白，确认音色对路再批量合成，避免浪费配额。

#### 反例 vs 正例

**反例**：`ai_voices` 表空 → 系统兜底分配了 `nova`（OpenAI），但 TTS 走 minimax → 全集对白合成失败，`composeError` 一片红。

**正例**：先录入 minimax 音色 → 每个 character 的 `voiceStyle` 都指向真实 voiceId → 试听一句「你走吧」确认情绪 → 批量合成。

### 3. 分镜：怎么判断对不对、怎么增删改

#### 判断标准清单

- [ ] **镜头数合理**：30 秒场景拆 2–3 镜；一集 60 秒 ≈ 4–6 镜。
- [ ] **单镜时长**：默认 10–15s 太拖。**短视频建议改 systemPrompt 为「每镜 4–6 秒」**（设置 → Agent 配置 → storyboard_breaker）。注意 `save_storyboards` 默认 10（`sb.duration || 10`）。
- [ ] **总时长**：`duration` 之和 ≈ episode 目标时长（系统自动算 `episodes.duration = ceil(totalDuration/60)`）。
- [ ] **景别 / 角度 / 运镜搭配**：远景交代环境 → 中景对话 → 近景/特写情绪。别整集都是中景。
- [ ] **`character_ids` 正确**：每个出场角色都绑定，且**必须来自本集已关联角色**（`validateStoryboardBindings` 不符直接抛错，整批不存）。
- [ ] **`sceneId` 正确**：location+time 能匹配 scenes 表，否则抛错。
- [ ] **首帧钩子**：第 1 镜 `imagePrompt` 必须有视觉冲击力（决定观众是否划走）。
- [ ] **`video_prompt` 有 `<n>` 分段**：每 3 秒一段，标签 `<location>/<role>/<voice>/<n>`。
- [ ] **`description` ≠ `video_prompt`**：前者人读，后者模型读，不能复用。
- [ ] **对白与镜头对应**：`dialogue` 里的角色必须在 `character_ids` 里。

#### 调整方法

| 症状 | 怎么改 |
|---|---|
| 单镜时长不对 | 前端编辑该镜 `duration`；或对话框「把第 3 镜 duration 改 5」 |
| 改 `image_prompt` | 前端编辑 `storyboards.imagePrompt`，重出该镜首帧 |
| 改 `video_prompt` | 前端编辑 `storyboards.videoPrompt`，重出该镜视频 |
| 整体重拆节奏 | 改 `agent_configs.storyboard_breaker.systemPrompt` 加「每镜 4–6 秒，节奏快，首镜必须有视觉钩子，景别要远近交替」 |
| 增删镜头 | 前端编辑；注意 `save_storyboards` 重跑会**先删光本集所有分镜再重存**（`storyboard-tools.ts:190`），不可撤销，重跑前先备份 |
| `character_ids` 报错 | 检查角色是否已关联到本集，没关联先重跑 extractor |

#### 反例 vs 正例

**反例 video_prompt**：「小明在咖啡厅等小红，两人见面。」——没分段、没标签，模型乱拍。

**正例 video_prompt**：
```
0-3秒：<location>咖啡厅角落</location>，近景，<role>小明</role>低头看手机，眉头微皱。<n>
3-6秒：中景，<role>小红</role>推门走入，风铃轻响。<n>
6-9秒：过肩镜头，两人对视，小红微笑落座。
```

**反例首镜**：「空镜头，街道远景。」——观众第一秒划走。

**正例首镜**：主角特写，雨水顺着左眉骨疤痕滑下，眼神阴郁——视觉钩子。

### 4. 角色形象：怎么调出满意效果

#### 外观描述要写哪些要素

固定结构（缺一不可）：
1. **基本**：性别、年龄、身高、体型。
2. **面部锚点**：脸型、眉毛、眼睛（单/双眼皮）、鼻、嘴、疤痕 / 痣 / 纹身（位置精确到 cm）。
3. **发型**：长度、颜色、造型。
4. **着装**：颜色、款式、材质、层次。
5. **配饰**：手表、耳钉、项链、眼镜（位置精确）。
6. **气质**：2–3 个形容词。

#### 风格锚

角色图 prompt 固定加 `cinematic portrait, consistent art style`。要全局统一画风，在 `appearance` 末尾加风格短语，如「cinematic film still, moody lighting, Kodak 250D」——所有角色都带这一句，画风就锁住。

#### 重抽策略

- 单角色不满意 → 直接重新点「生成」按钮，新图覆盖 `imageUrl`。
- **不要反复重抽同一字段**，超过 3 次还崩 = 改 `appearance`，不是靠运气。
- 主角图必须先满意，再铺配角——配角可以参考主角画风微调。

#### 多人外观差异化降串脸

群像剧本最大坑是角色互相串脸。手法：
1. **每个角色强差异化锚点**：主角「左眉骨 3cm 疤」、配角「右脸酒窝 + 金丝眼镜」、反派「光头 + 龙纹脖链」。脸部锚点越多越不串。
2. **发色 / 着装撞色**：不要全员黑西装。颜色区分最稳。
3. **体型差异**：胖瘦高矮拉开。
4. **参考图复用**：主角先出一张满意的图，作为后续每镜生成的参考图（见第 8 节）。

#### size

跟随 provider 默认（角色图通常竖屏或方屏，单镜不传 size）。要锁竖屏 9:16，在 provider 支持的尺寸里选竖屏档。

---

### 5. 场景图：怎么调

#### 判断标准

- [ ] **`prompt` 英文 + 纯背景 + 氛围词**：固定加 `cinematic scene, atmospheric lighting`。
- [ ] **整集视觉基调统一**：所有场景描述带同一个色调短语（如 `warm tungsten, neon reflections, moody night`）。
- [ ] **同地点不同时段视觉区分**：黄昏暖橙 vs 深夜冷蓝，prompt 里要写死光线。
- [ ] **竖屏 9:16**：在 provider 尺寸档里选竖屏（如 `540x960` 或对应比例）。

#### 调整方法

- 改 `scenes.prompt` 加氛围短语：例「rainy night, neon signs reflecting on wet asphalt, warm tungsten lamp inside, no people, moody cinematic」。
- 整集色调不统一 → 批量在每场景 prompt 加同一个色调锚。
- **传播规则**：场景图更新不会自动刷新已生成的镜头图，要手动重生镜头图才用上新场景参考。

#### 反例 vs 正例

**反例**：「咖啡厅。」——平淡白光，每张场景图风格漂移。

**正例**：「rainy night coffee shop, neon signs through window, warm tungsten desk lamp, wet street reflection, moody cinematic, kodak film grain」——氛围锁死。

---

### 6. 配音：试听 + 时长对齐

#### 判断标准

- [ ] **音色情绪对路**（试听一句确认）。
- [ ] **TTS 时长 ≈ 视频时长**：compose 用 `-shortest` 硬截，TTS 比视频长 → 丢尾句；TTS 比视频短 → 尾段静音黑场。
- [ ] **字幕与对白对齐**：字幕用 `duration-1` 粗估结束点，与真实音频无关，可能「字没了声音还在」。

#### 调整方法（关键：宁可重生成，勿硬裁）

- **改文本调速**：TTS 太长 → 拆句 / 精简台词；TTS 太短 → 合并 / 加气口。
- **换 voice 调速**：换音色 / 调 `voice_setting.speed`。
- **清旧 `ttsAudioUrl` 再 compose**：`composeStoryboard` 会复用已存音频，改了文本不清旧音频不会重生成。
- **字幕**：白字 FontSize=20、黑描边 Outline=2（compose 烧死），merge 不再叠加。要改字幕样式需改 `ffmpeg-compose.ts` 源码（非配置项）。
- **BGM**：系统不内置，要加需外挂 `-i` 输入手动合入（现成代码无此路径）。

#### 反例 vs 正例

**反例**：TTS 8 秒、视频 5 秒 → `-shortest` 把后 3 秒台词截掉，观众听到一半消失。

**正例**：拆台词成两句、每镜 5 秒 → TTS ≈ 4.5 秒，对齐视频。

### 7. 镜头图·宫格图排版策略

> 本节只讲「怎么选」与「怎么判断」。完整操作步骤（每步点哪个按钮）见[第一节·第 10 步](#第-10-步镜头图片制作镜头图片-tab--含宫格图工具完整走法)。

#### 三种模式选哪个

| 你的目标 | 选哪个 | 布局 | 镜头选择 |
|---|---|---|---|
| 整集一次性铺多镜头首帧（**推荐起步**） | `first_frame` | 3x3（≤9 镜）/ 4x3（≤12 镜） | 多选，上限=格数 |
| 单镜要高质量运镜补间 | `first_last` | **强制 2 列**（2x3 / 2x4） | 多选 3–4 镜 |
| 关键镜头补参考图 | `multi_ref` | 2x2（4 张，**别开大**） | **单选**一个镜头 |

#### 两个独立开关（再强调）

- **`gridMode`**（宫格工具顶部 Tab）：控制批量出图时每格是什么。只在「用宫格出图」时生效。
- **`frameMode`**（镜头列表顶部下拉 `first` / `first_last`）：控制镜头列表 UI 和单镜生成行为 + 视频参考模式。

#### 推荐组合

- 起步：`gridMode=first_frame` + `frameMode=first` → 一次铺所有首帧，视频走 `single`。
- 进阶：先铺好首帧，再 `frameMode=first_last`，对需要运镜的镜头单独点尾帧按钮（单镜走 `/images/generate`，不经过宫格）。
- 关键镜头强化：`gridMode=multi_ref` + 2x2 → 4 张不同机位，分配为 `reference`。

#### 首尾帧接力怎么做（判断要点）

1. 先用 `first_frame` 把首帧铺好。
2. 挑转身 / 走动 / 推门 / 特写变化的镜头（判断依据：`action` / `movement` 字段有连续动作）。
3. `gridMode=first_last`，**布局 2 列**，多选 3–4 镜。
4. 生成 → 切分 → **偶数格（index 0,2,4…）分配为 `first_frame`，奇数格（1,3,5…）分配为 `last_frame`**（严格按奇偶，否则视频把尾帧当首帧）。
5. **前提**：分镜里 `action` / `movement` 字段要填具体，否则尾帧只是换个姿势的静帧，没有动作连续性。

#### 6 张参考上限下多角色怎么选 subset

参考图收集顺序固定（`grid.ts:78`）：镜头首帧 → 镜头尾帧 → 镜头图 → 镜头参考图 → 场景图 → 角色图，到 6 张截断。**多角色群像剧本极易爆 6 张，爆了靠后的角色/场景图被丢，模型只能凭名字猜脸 → 一致性崩**。

策略：
1. **主角先入库**：顺序按 DB id 升序，让主角 `imageUrl` 更早生成（更早入库）。
2. **清无用参考**：把不要的镜头 `referenceImages` 清空、未用场景 `imageUrl` 删掉，腾配额。
3. **场景优先时删角色图**：场景一致性更重要就先删部分角色图（场景图在角色图前）。
4. **multi_ref 别开大**：一个镜头出 9 张瞬间吃满，后续该镜头视频生成只剩这 9 张（截到 6），角色 / 场景参考全丢。

#### 切分后格子怎么正确分配

- **先对位再分配**：看每格画的是哪个镜头，再去镜头列表找对应 id，**不要按格顺序硬塞**（模型不一定按你给的镜头顺序画）。
- **first_last 严格奇偶**：偶数格首帧、奇数格尾帧，否则补间错乱。
- **一格画了两个镜头 / 粘连** → 不分配，重出整图（像素等分切不开粘连）。
- **一格画空 / 出现文字水印** → 不分配，重出整图（单格不能单独重生）。
- **重出整图**：回到 step 0，改 `custom_prompt` 加强 `no merged panels, exactly N visible panels, no text`，或换 provider。
- **cell prompt 改英文**：代码生成的 cell prompt 是中文「格1：」混合，**图片模型英文效果更好**，生成后手动改英文。

### 8. 一致性实战

#### 主角锁脸

- 主角 `appearance` 写 300+ 字，脸部锚点精确到疤痕位置 / 痣。
- 主角先出一张满意图，作为**后续每镜生成的参考图**（写入 `referenceImages`）。
- 6 张参考配额里，主角那张必须占一个位（顺序靠前）。

#### 跨镜参考图复用

- `first_frame` 宫格模式跨镜一致性最好（同一张大图里人脸 / 服装 / 场景统一），**整集首帧首选此模式**。
- 主角首帧定下后，所有该主角出现的镜头都把这张首帧拉进参考图。

#### 群像泛化

- 每个角色强差异化锚点（见第 4 节）。
- 群像镜头建议先 multi_ref 出该镜头的 4 张参考，让模型有多个构图线索。
- 不要指望一次出图所有人脸都对，群像镜后期靠人工微调单角色。

#### 记录重抽次数当崩坏率

- 每个角色 / 每镜记下「重抽了几次才满意」。某角色重抽 ≥5 次还崩 = `appearance` 写得不够锚定，回去补字段，不是靠重抽。
- 某镜头反复崩 = 该镜头 `image_prompt` 太模糊，回去精修。

#### 何时考虑补 LoRA

- 当前系统**未集成 LoRA 训练 / 加载**，所有一致性靠 prompt + 参考图。
- 当出现：主角重抽 ≥10 次仍串脸、群像镜头几乎全员崩、跨集主角脸持续漂移 → 参考图 + prompt 这套已经到天花板，应考虑外挂训练 LoRA（需自行接入，非系统内置）。

### 9. 成本控制

#### 视频 = 大头（85%+）

- 视频生成是整条流水线最贵环节，图片 / TTS 成本相对可忽略。
- **铁律：先小子集验证再批量**。挑 2–3 个代表镜（一个对话镜 + 一个运镜镜 + 一个特写镜）跑通整链路，确认质量后再铺全集。

#### 重抽预算

- 每镜视频重抽 ≤2 次。超 2 次还崩 = 上游（首帧 / prompt / 参考图）有问题，回去修上游，不是堆视频配额。
- 图片重抽便宜，可以多试；视频贵，省着用。

#### fast / pro 档位选择

- **Seedance** 默认 `doubao-seedance-1-5-pro-251215`，duration 归一到 [4,12]s。
- 验证阶段：用 lite / fast 档（如 Seedream lite、wan2.6-flash、doubao-seedance-1-0-pro-fast）快速迭代 prompt。
- 出片阶段：切 pro 档（Seedance pro、speech-2.8-hd TTS）保质量。
- **TTS 只 minimax 一家**，无档位可选，model 默认 `speech-2.8-hd`。

#### 其他省钱点

- 宫格一次出多图 < 单镜逐张出图（成本 + 一致性双赢）。
- 参考图保持 ≤6 张且精挑（超出被截断浪费配额）。
- `referenceMode='none'`（默认缺省值）= 纯文生视频，质量最差但最便宜——验证 prompt 时可用，出片别用。

## 三、模型选型推荐（官方 API，禁中转站）

### 总则：怎么把模型接进系统

本系统是一个**「OpenAI 兼容文本客户端 + provider adapter 注册表」**的架构。两类服务的接入规则不同：

- **文本（LLM）**：走 `backend/src/services/ai.ts`，是标准 OpenAI SDK。它按 `provider` 字段自动拼接 endpoint 前缀（`ai.ts:18 getTextProviderBaseUrl`）：`provider=openai/openrouter/chatfire` 拼 `/v1`、`provider=volcengine` 拼 `/api/v3`、`provider=ali` 拼 `/api/v1`、其余原样使用。
- **图像/视频/TTS**：走 `backend/src/services/adapters/registry.ts` 注册表。`provider` 名必须命中注册表里已注册的键，否则 `getXxxAdapter()` 会**静默 fallback 到 minimax**（即「设置页能保存，但请求全打到 minimax 接口，必报错」）。

在 **Web 设置页（settings.vue）**，四类服务分别填四项：
1. `provider`（必须在注册表里：文本任意，图像可选 `openai/gemini/minimax/volcengine/ali`，视频可选 `minimax/volcengine/vidu/ali`，TTS 仅 `minimax`）
2. `baseUrl`（官方域名，不要带尾部斜杠，系统会按上表拼接前缀）
3. `apiKey`（官方 Key；火山方舟用 `ep-` 开头的 **接入点 ID** 而非 AK/SK）
4. `model`（官方模型名）

**用户明确禁用中转站**：火宝官方预设的 `api.chatfire.site` + `gemini-3-pro-preview` 路径不采用；所有推荐均为厂商官方 endpoint。

成本占比事实（来自 `docs/short-drama-research/02-huobao-deep-dive.md §2.2`）：单集 10 镜约 41 元基线中，**视频占 85%+、TTS<1%、LLM<1%**。所以文本/TTS 选型优先看「质量+接入顺滑」，视频是唯一需要严肃算账的环节。

### 诚实标注（接入限制）

1. **TTS 目前仅 minimax adapter**：`registry.ts` 的 `ttsAdapters` 只有 `minimax` 一个键，CosyVoice/豆包语音/Azure/ElevenLabs 均需在 `backend/src/services/adapters/` 下新建 `xxx-tts.ts` 并在 registry 注册（参考 minimax-tts.ts 结构）。
2. **视频厂商未内置 adapter**：**可灵（Kling）、Veo、即梦、混元** 等均未注册 adapter，需新写 `xxx-video.ts` + registry 注册 + settings.vue 加 provider 下拉项。填了未注册的 provider 名会被静默 fallback 到 minimax，表现为「能保存但请求必报错」——这是最易踩的坑。
3. **图像厂商未内置 adapter**：**可灵图像、即梦、混元图像** 等同上需新增 adapter。
4. **智谱 GLM**：官方端点 `/api/paas/v4` 与系统 `ai.ts` 自动拼 `/v1` 的逻辑冲突，需小改源码或前端自定义 provider，新手不建议。
5. **Anthropic / Google 原生 LLM**：非 OpenAI 协议，本系统不直接支持，需经第三方转售（已禁用）或自建 adapter。
6. **Vidu 全家**：技术上 adapter 已注册但 webhook+Token 鉴权对新手是雷区（issue #88 丢任务记录），非必要不用。

## 结语：给零基础新手的执行顺序建议

1. **先填设置页（治本第一步）**：按[第三节·综合套件](#综合套件推荐新手首选质量成本接入顺滑三平衡)把 4 类官方 AI 服务（deepseek-chat / Seedream+万相 / Seedance 1.5-pro / MiniMax speech-2.8-hd）配齐，并把 minimax 官方音色录进 `ai_voices` 表。这一步没做好，后面全是错配（#85）。
2. **精修 `characters.appearance`**：每个角色写到 300+ 字、脸部锚点精确到疤痕/痣位置。这是全流程最该手动投入时间的字段，直接决定角色图一致性。
3. **分镜调短、首镜做钩子**：把 `storyboard_breaker` 的 systemPrompt 改成「每镜 4–6 秒，节奏快，首镜必须有视觉钩子，景别远近交替」，避免默认 10–15s 拖沓。
4. **宫格起步用 `first_frame` 一次性铺首帧**：`gridMode=first_frame` + `frameMode=first`，整集首帧跨镜一致性最好；需要运镜的镜头再单独补 `first_last` 尾帧（注意奇偶格分配）。
5. **每步跑完先抽查 1–2 条**：别等整集跑完发现全错。视频是 85%+ 成本，挑 2–3 个代表镜（对话镜/运镜镜/特写镜）先跑通整链路，确认质量后再铺全集；单镜失败别整集重跑，看 `composeError` / `prod-error` 真实错误逐镜重试（#84）。
